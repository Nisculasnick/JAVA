Classes e Métodos em Java

1. Classe

A classe é como uma planta ou um molde. Ela define a estrutura e o comportamento que os objetos terão. É o "tipo" que você cria.

Pense nela como a receita para fazer um bolo de chocolate: ela diz quais ingredientes (atributos) e quais passos (métodos) são necessários para fazer um bolo.

Exemplo (Classe Financiamento)


public class Financiamento { // Esta é a CLASSE "Financiamento"
    // ... corpo da classe ...
}

public class InterfaceUsuario { // Esta é a CLASSE "InterfaceUsuario"
    // ... corpo da classe ...
}

public class SimuladorFinanciamento { // Esta é a CLASSE "SimuladorFinanciamento" (seu Main)
    // ... corpo da classe ...
}

2. Atributo

Um atributo (ou variável de instância) é uma característica ou propriedade de um objeto da classe. Ele guarda um dado específico para cada objeto que for criado.

No exemplo do bolo, seriam os "ingredientes" como a quantidade de farinha, ovos, açúcar.

O modificador private antes do tipo do atributo significa que ele só pode ser acessado (lido ou modificado) de dentro da própria classe, o que ajuda no encapsulamento (proteção dos dados).

O modificador final indica que o valor desse atributo será definido apenas uma vez (normalmente no construtor) e não poderá ser alterado depois.

Exemplo (Atributos da Classe Financiamento)


class Financiamento {
    private final double valorImovel;     // Atributo: guarda o valor do imóvel para este financiamento.
    private final int prazoFinanciamento; // Atributo: guarda o prazo em anos para este financiamento.
    private final double taxaJurosAnual;  // Atributo: guarda a taxa de juros anual para este financiamento.
    // ...
}

class InterfaceUsuario {
    private Scanner scanner; // Atributo: o objeto Scanner usado por esta InterfaceUsuario.
    // ...
}

3. Construtor

O construtor é um método especial que tem o mesmo nome da classe e NÃO tem tipo de retorno (nem mesmo void). Ele é chamado automaticamente quando você cria um novo objeto usando a palavra-chave new.

Sua função principal é inicializar os atributos do objeto recém-criado, garantindo que ele nasça em um estado válido.

Ele pega os parâmetros (valores que vêm de fora) e os usa para definir os atributos internos do objeto (usando this. para diferenciar).

Exemplo (Construtor da Classe Financiamento)


class Financiamento {
    private final double valorImovel;
    private final int prazoFinanciamento;
    private final double taxaJurosAnual;

    // CONSTRUTOR:
    public Financiamento(double valorImovel, int prazoFinanciamento, double taxaJurosAnual) {
        this.valorImovel = valorImovel;         // Inicializa o atributo 'valorImovel'
        this.prazoFinanciamento = prazoFinanciamento; // Inicializa o atributo 'prazoFinanciamento'
        this.taxaJurosAnual = taxaJurosAnual;   // Inicializa o atributo 'taxaJurosAnual'
    }
    // ...
}

// Como ele é chamado no método main:
// Financiamento financiamento = new Financiamento(valorImovel, prazoFinanciamento, taxaJurosAnual);
// 'new Financiamento(...)' chama o construtor acima.

4. Método

Um método é um bloco de código que executa uma tarefa específica. Ele define um comportamento ou uma ação que os objetos da classe podem realizar.

Pode receber parâmetros (valores de entrada) e/ou retornar um valor.

No exemplo do bolo, seria o "passo para assar o bolo" ou o "passo para decorar o bolo".

Exemplo (Métodos da Classe Financiamento e InterfaceUsuario)


class Financiamento {
    // ... construtor e atributos ...

    // MÉTODO: calcula o pagamento mensal. Retorna um 'double'.
    public double calcularPagamentoMensal() {
        return (valorImovel / (prazoFinanciamento * 12)) * (1 + (taxaJurosAnual / 12));
    }

    // MÉTODO: calcula o total do pagamento. Retorna um 'double'.
    public double calcularTotalPagamento() {
        return calcularPagamentoMensal() * prazoFinanciamento * 12;
    }
}

class InterfaceUsuario {
    // ... atributo scanner e construtor ...

    // MÉTODO: pede e retorna o valor do imóvel. Não recebe parâmetros.
    public double pedirValorImovel() {
        System.out.print("Digite o valor do imóvel: ");
        return scanner.nextDouble();
    }

    // MÉTODO: pede e retorna o prazo do financiamento. Não recebe parâmetros.
    public int pedirPrazoFinanciamento() {
        System.out.print("Digite o prazo do financiamento em anos: ");
        return scanner.nextInt();
    }

    // MÉTODO: pede e retorna a taxa de juros. Não recebe parâmetros.
    public double pedirTaxaJurosAnual() {
        System.out.print("Digite a taxa de juros anual (em decimal, ex: 0.05 para 5%): ");
        return scanner.nextDouble();
    }
}

5. Objeto (Instância)

Um objeto é uma "materialização" de uma classe. É a "coisa" concreta criada a partir da planta/molde da classe. Você pode criar vários objetos da mesma classe, e cada um terá seus próprios valores para os atributos.

No exemplo do bolo, seria o "bolo de chocolate específico" que você assou.

É criado usando a palavra-chave new e o construtor da classe.

Exemplo (Criação de Objetos no Main)


public class SimuladorFinanciamento {
    public static void main(String[] args) {
        // Cria um OBJETO da CLASSE InterfaceUsuario:
        InterfaceUsuario interfaceUsuariovar = new InterfaceUsuario(); 

        // Cria um OBJETO da CLASSE Financiamento, usando os valores lidos:
        Financiamento financiamento = new Financiamento(valorImovel, prazoFinanciamento, taxaJurosAnual);
    }
}
